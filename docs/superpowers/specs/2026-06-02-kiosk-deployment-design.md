# Kiosk Deployment — оффлайн-разворачивание сайта «Путешествуй в КЧР» на выставочном ноутбуке

**Дата:** 2026-06-02
**Статус:** draft → на ревью

## 1. Контекст

На выставке к ноутбуку подключают touch panel. На этом ноутбуке должен крутиться полный сайт «Путешествуй в КЧР» (фронт + бэк + БД + изображения) в **полностью оффлайн** режиме. Разворачивает не-программист.

Существующие проекты:
- Бэкенд: `C:\git back\puteshestvuy_v_kchr_back` — Node.js + Express + Prisma + MongoDB.
- Фронт: `C:\github\puteshestvuy_v_kchr` — Vite + React + MUI.
- Архив `uploads/` со всеми изображениями.

**Жёсткое ограничение:** в исходниках обоих проектов ничего не правим. Все артефакты для выставочной сборки — новые файлы в отдельной папке.

## 2. Цели и не-цели

**Цели**
- Один раз запустить `install.bat` в офисе с интернетом → ноутбук готов.
- На выставке — клик по иконке «Путешествуй в КЧР» на рабочем столе → сайт открывается в kiosk-режиме.
- Все интерактивные фичи, требующие интернета (карты, бронирование, обратная связь, регистрация, админка), визуально скрыты для посетителя.
- Контент (места, маршруты, новости) — реальные данные с продакшен-машины.
- Обновление контента: один скрипт у автора → перевыгрузка дампа.

**Не-цели**
- Не делаем portable-сборку без интернета на этапе установки (интернет при установке будет — подтверждено).
- Не делаем поддержку macOS/Linux. Только Windows 10/11.
- Не правим исходники backend/frontend.
- Не строим CI/CD, обновлений по сети, телеметрии.

## 3. Архитектура

Один ZIP-архив, разворачивается в `C:\puteshestvuy-kiosk\`. Внутри:

```
C:\puteshestvuy-kiosk\
├── install.bat                  # запускается один раз в офисе с интернетом
├── start.bat                    # запускается по иконке на рабочем столе
├── stop.bat                     # аккуратное выключение
├── README-для-установки.txt     # 5-строчная инструкция для не-программиста
├── kiosk-launcher.html          # обёртка с iframe + CSS-инъекция
├── env-templates\
│   ├── backend.env              # копируется в backend\.env при установке
│   └── frontend.env             # копируется в frontend\.env при установке
├── db-dump\                     # mongodump архив с актуальным контентом
├── uploads-archive\
│   └── archive_02062026_1620.zip  # архив с изображениями (имя может меняться — установщик берёт первый archive_*.zip)
├── uploads\                     # сюда install.bat распаковывает архив выше
├── backend\                     # копия исходников бэкенда (без node_modules / .git)
├── frontend\                    # копия исходников фронта (без node_modules / .git)
└── scripts\
    ├── make-dump.bat            # запускается на машине автора для свежего дампа
    └── refresh-kiosk.bat        # переустановка контента на выставочной машине
```

Бэкенд после установки слушает `http://localhost:4000`, фронт собран в `frontend\dist` и раздаётся `npx serve` на `http://localhost:3000`. Chrome открывает `file:///C:/puteshestvuy-kiosk/kiosk-launcher.html` в kiosk-режиме; launcher грузит сайт во встроенный iframe и поверх инжектит CSS, скрывающий лишние элементы.

## 4. install.bat — что делает

Запускается ровно один раз в офисе с интернетом. Шаги:

1. **Проверка/установка Node.js 20 LTS** — если `node --version` < 20, тихо скачивает MSI с nodejs.org и ставит.
2. **Проверка/установка MongoDB Community 7.x** — если MongoDB не установлен как сервис, скачивает MSI с mongodb.com и ставит с автозапуском Windows-сервиса.
3. **Установка зависимостей бэка**: `cd backend && npm install`.
4. **Prisma**: `npx prisma generate && npx prisma db push` (создаёт коллекции в локальной MongoDB).
5. **Восстановление дампа**: `mongorestore --archive=..\db-dump\dump.archive --nsInclude=puteshestvuy.*`.
6. **Распаковка uploads**: ищет в `uploads-archive\` первый файл `archive_*.zip` и распаковывает в `uploads\` через PowerShell `Expand-Archive` (входит в Windows 10/11 «из коробки»). Если в `uploads\` уже есть файлы — пропускает шаг (идемпотентность).
7. **Привязка uploads к бэкенду**: создаёт junction `backend\uploads` → `..\uploads` через `mklink /J` (не требует прав админа). Если не сработает — fallback на копирование.
8. **`.env` файлы**: копирует `env-templates\backend.env` → `backend\.env`, `env-templates\frontend.env` → `frontend\.env` **до** шага сборки фронта (Vite запекает переменные в build).
9. **Сборка фронта**: `cd frontend && npm install && npm run build`. На выходе `frontend\dist`.
10. **Ярлык на рабочем столе** — через `powershell New-Object -ComObject WScript.Shell`. Цель ярлыка: `start.bat`, иконка: своя ico (положим в архив).
11. **Финальное сообщение**: «Установка завершена. Теперь можно отключать интернет и везти ноутбук на выставку».

**Идемпотентность**: повторный запуск `install.bat` безопасен — пропускает шаги, где артефакты уже на месте. Дамп **накатывается с `--drop`**, чтобы свежий контент полностью заменял старый.

### Содержимое env-файлов

`env-templates\backend.env`:
```
NODE_ENV=production-local
DATABASE_URL=mongodb://localhost:27017/puteshestvuy
JWT_SECRET=kiosk-local-only-secret-can-be-anything
# SMTP не настраиваем — формы будут «fail silently», что нам и нужно
```

`env-templates\frontend.env`:
```
VITE_API_URL=http://localhost:4000/api
VITE_YANDEX_MAPS_API_KEY=
```

`NODE_ENV=production-local` — НЕ `production`, иначе `server.js` потребует SSL. Это `dev`-подобный режим, но без morgan-логов в консоль. Если есть сомнения — оставим `development`; сейчас бэк уже корректно работает в этом режиме.

## 5. start.bat — что делает

Запускается с рабочего стола по иконке. Шаги:

1. **MongoDB** уже работает как Windows-сервис (запускается с системой) — не трогаем.
2. **Бэкенд**: `start /MIN cmd /c "cd backend && node server.js"`. Окно сворачивается, бэкенд слушает 4000.
3. **Sleep 2 секунды** — дать бэку подняться.
4. **Раздача фронта**: `start /MIN cmd /c "npx serve frontend\dist -p 3000 -s --no-clipboard"`. `-s` включает SPA-fallback (любой роут отдаёт `index.html`).
5. **Sleep 1 секунда**.
6. **Chrome в kiosk-режиме**:
   ```
   start chrome --kiosk --app=file:///C:/puteshestvuy-kiosk/kiosk-launcher.html ^
     --disable-pinch --overscroll-history-navigation=0 ^
     --noerrdialogs --disable-session-crashed-bubble ^
     --user-data-dir=C:\puteshestvuy-kiosk\chrome-profile
   ```
   Отдельный профиль (`--user-data-dir`) гарантирует чистый запуск без зависших табов от пользователя.

**Авто-перезапуск Chrome** (опционально, на случай случайного Alt+F4): в `start.bat` обернём запуск Chrome в цикл `:loop` — закрылся, запустился снова. Реализуем как опцию; включается флагом в начале файла.

**stop.bat**: `taskkill /F /IM chrome.exe /IM node.exe`.

## 6. Скрытие фич через kiosk-launcher.html

`kiosk-launcher.html` — единственное, что Chrome открывает. Внутри iframe на `http://localhost:3000` + JS, который при загрузке каждого роута дописывает в `<head>` iframe-документа `<style>` со скрывающими правилами.

Поскольку iframe и сайт оба на `localhost`, same-origin policy не мешает доступу к `iframe.contentDocument`.

**Что прячем (финальные селекторы подбираем во время реализации, посмотрев актуальный билд фронта):**
- Карты Yandex — контейнеры компонентов `YandexMapRoute`, `YandexMapPlace`, `YandexMapPicker`.
- Формы бронирования и обратной связи в футере.
- Ссылки/кнопки логина, регистрации, профиля.
- Ссылки на `/admin`.

**Реактивность роутов**: SPA меняет содержимое без перезагрузки iframe, поэтому правила инжектим один раз при первом `load` iframe — глобальный `<style>` в `<head>` остаётся при SPA-навигации.

**Что НЕ прячем кодом, потому что и так не работает корректно:**
- Карты сами не загрузят тайлы без интернета — пустой div. CSS-скрытие — косметика.
- SMTP не настроен → заявки бронирования и фидбека просто записываются в локальную БД, письма не уходят (это уже текущее поведение бэка, см. `app/utils/mailer.js`). Но кнопки прячем CSS, чтобы посетитель не нажимал.
- Админка: на выставочной машине **не запускаем** `npm run create-admin` → SUPERADMIN отсутствует, войти физически некому. Ссылку прячем CSS.

## 7. Обновление контента после установки

Сценарий: на выставке исправили опечатку в описании места у себя дома → надо подтянуть на выставочный ноут.

**На машине автора:**
```
scripts\make-dump.bat
```
Делает `mongodump --db=puteshestvuy --archive=dump.archive`, кладёт в текущую папку. Дальше автор копирует `dump.archive` на флешку, везёт на выставку, кладёт в `C:\puteshestvuy-kiosk\db-dump\` и запускает:

**На выставочной машине:**
```
scripts\refresh-kiosk.bat
```
Делает `mongorestore --drop --archive=...\db-dump\dump.archive`. Сайт перезапускается автоматически (или хватит F5 в браузере).

Для обновления изображений — копирует свежий `uploads/` поверх существующего.

## 8. Риски и mitigation

| Риск | Mitigation |
|---|---|
| MongoDB MSI требует прав администратора → не-программист может не пройти UAC | `install.bat` запускается с правами админа (через `runas` или просто двойной клик из админ-аккаунта). Это документируем в README. |
| Vite запекает `VITE_API_URL` в build → если env-файл не на месте до сборки, фронт будет смотреть на dev-сервер из существующего `.env` | `install.bat` копирует `frontend.env` → `frontend\.env` **строго перед** `npm run build`. Существующий `.env` в исходниках перезаписывается локальной копией. |
| Chrome обновится и сломает флаги kiosk | Maловероятно, флаги стабильны годами. На выставочной машине обновления Chrome можно отключить, но в дизайн это не входит. |
| Yandex Maps SDK при загрузке тайлов делает запросы и зависает на ожидании DNS | CSS прячет контейнер до того, как карта успеет начать инициализацию (правило `display: none` блокирует исполнение скриптов карты внутри). |
| Пользователь случайно вышел из kiosk через Alt+F4 | Опциональный авто-перезапуск Chrome в `start.bat` (цикл `:loop`). |
| Дамп несовместим со схемой Prisma (если автор успел поменять схему) | `install.bat` делает `prisma db push` **до** `mongorestore`. MongoDB schemaless — `db push` создаёт индексы и валидаторы, `restore` льёт документы. Если структура документов в дампе несовместима — упадёт `restore`. Документируем: «менял схему — пересобери дамп». |

## 9. Открытые вопросы

- Иконка для ярлыка (`.ico`) — нужна или сойдёт дефолтная Chrome-иконка? **По умолчанию: дефолтная**, на ревью спроси если хочешь своя.
- Авто-запуск при включении ноутбука (ярлык в `shell:startup`) — **по умолчанию: НЕ включаем**, человек кликает иконку. Если потом захочешь — добавим одной строкой в install.bat.
- Перезапуск-цикл Chrome (защита от Alt+F4) — **по умолчанию: включаем**. Безопаснее.
